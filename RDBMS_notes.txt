Database : Collection of files & programs which allows the user to access files (Add, inserting, deletion, modify/change)

Benefits : 

Users : DBA, Designer, Developer, End user

Database model : Blue print of database functionality (storage,organised & manipulated)

ERD model

RDBMS : IBM"s DB2, Oracle, Mysql, Maria DB --> SQL

Schema : Description of the Database

1. External/view level --> Individual user level view

2. Conceptual level --> Common user level view

3. Internal level --> Storage view

RDBMS : 
-----------

Relations --> Tables ( 2 dimensional)

Attributes(columns) and tuples(rows)

Keys
------

Candidate key : Attribute to distinguish the relationship (ID, contact num, email ID)

Key Attributes  & Non-key attributes

Primary Keys : Unique & Cannot be NULL

Composite Key : Combination of more than one attribute)

Entity Instance : One entry

Degree of Relationship (cardinality) : An association between 2 or more entities
